# 风险与架构决策记录

## ADR-001：使用 Android Native Kotlin + CameraX

状态：接受。

原因：本项目核心依赖相机、传感器、MediaStore、EXIF、定位等 Android 原生能力。Native Kotlin 控制力强，长期维护风险低。

不选 Flutter/RN 的原因：跨平台收益不大，且相机/传感器/存储细节需要插件桥接，兼容问题更多。

## ADR-002：使用 MediaStore 保存照片

状态：接受。

原因：符合 Android scoped storage，保存后系统相册可见。Android 10+ 通常不需要传统写存储权限。

风险：不同 Android 版本行为存在差异，需要真机测试。

## ADR-003：水印本地 Canvas 渲染

状态：接受。

原因：无服务端成本、隐私好、离线可用、适合静态图片水印。

风险：需要处理 EXIF 方向、大图内存、横竖图布局。

## ADR-004：天气后置且可降级

状态：接受。

原因：天气依赖网络和外部 API，不应影响拍照主链路。MVP 先不阻塞在天气上。

## ADR-005：MVP 单 module，清晰 package 边界

状态：接受。

原因：项目初期更快落地；通过 feature/domain/data/core/design 分包维护边界。后续复杂后再拆 Gradle module。

## 主要风险

### 大图 OOM

缓解：拍照先落临时文件；渲染在后台线程；必要时限制水印图最大边长；避免同时持有多份大 Bitmap。

### EXIF 方向错误

缓解：读取 ExifInterface，绘制前统一旋正，保存水印图时输出正常方向。

### 定位慢/失败

缓解：相机页提前刷新定位；拍照时使用最近有效定位；短超时；失败仍可拍照。

### 方位不准

缓解：低精度时提示；可关闭方位字段；水印标注为“设备朝向”，不当作专业测量。

### 天气/地址 API 不稳定

缓解：短超时、缓存、可关闭、失败隐藏字段；API Key 不硬编码。

### logo Uri 权限失效

缓解：优先复制 logo 到 App 私有目录，或正确持久化 Uri 读取权限。

### 隐私风险

缓解：照片默认本地处理；不上传照片；EXIF GPS 默认关闭；天气查询可关闭；定位权限拒绝仍可拍照。
